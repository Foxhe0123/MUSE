#MUSE: Emulating Case-Based Writing Expertise via Multi-Agent LLMs for Long-Form Text Generation

 Writing Framework for Introductory Texts with LLMs
The use of large language models (LLMs) for article generation has become
 common practice. However, limited research has been conducted on writing intro
ductory texts, and existing methods often require substantial human involvement,
 leading to low efficiency and weak structural coherence. By integrating the case-based reasoning paradigm and emulating human article editing workflows, 
 we propose **MUSE**, a **M**ulti-agent **U**nified **S**tructured **E**xpository writing Framework. 
MUSE aligns the LLM-based multi-agent long-text generation process with the CAR paradigm, automatically generating and editing explanatory texts through three steps: planning, generation, and proofreading. Additionally, it incorporates
 reader agents and editor agents to iteratively refine the generated content. We con
duct comprehensive evaluations of generated articles using automatic and human
 evaluation. The results show that introductory texts generated by MUSE demon
strate certain advantages in both writing efficiency and overall quality compared to
 human-written and standard LLM-generated texts, with notable improvements in
 authenticity, accuracy, and coherence.

## MUSE Multi-Agent Automated Writing Framework

- 🔧 Model-agnostic: supports DeepSeek, GPT, Qwen, LLaMA (via config file)
- 📜 Outputs: `.md` and `.html` files with or without outlines and modification


## How to run

1. Clone this repository and install dependencies:

```bash
pip install -r requirements.txt
```

2. Prepare your configuration file config.json for API credentials:
```
{
  "deepseek": {
    "api_key": "your_deepseek_api_key",
    "base_url": "https://api.deepseek.com",
    "model": "deepseek-chat"
  },
  "gpt-4": {
    "api_key": "your_openai_api_key",
    "base_url": null,
    "model": "gpt-4"
  }
  // ... add other models
}
```

3. Use the command-line interface to run the generation pipeline:

```bash

python main.py \
  --task "Write an article about AI in education" \
  --model deepseek \
  --isEdit True \
  --edit_epic 3 \
  --isRead True \
  --isFeedback True
```

4. Output Files will be saved to the output/ directory in multiple formats:

* Title(model).md: full article

* Title(model)(without_Modify).md: pre-modification version

* Title(model)(without_outline).md: text-only version

* .html versions for each variant
